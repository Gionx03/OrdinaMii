
Formato usato:
METODO  /endpoint
Descrizione: cosa fa l'endpoint
Ruolo: ruolo che può usare l'endpoint


DISHES - PIATTI / MENU


GET /dishes
Descrizione: ottiene tutti i piatti oppure li filtra tramite query param.
Query param:
- category: filtra per categoria.
- name: cerca per nome.
-descr: cerca per la descrizione, che contiene ingredienti
Ruolo: tutti

GET /dishes/{id}
Descrizione: ottiene un singolo piatto tramite id.
Ruolo: tutti

POST /dishes
Descrizione: crea un nuovo piatto.
Ruolo: admin/staff

PUT /dishes/{id}
Descrizione: modifica un piatto esistente.
Ruolo: admin/staff

DELETE /dishes/{id}
Descrizione: elimina un piatto.
Ruolo: admin/staff


TABLES - TAVOLI


GET /tables
Descrizione: ottiene tutti i tavoli del ristorante.
Ruolo: tutti


POST /tables
Descrizione: crea un nuovo tavolo.
Ruolo: admin

PUT /tables/{id}
Descrizione: modifica un tavolo esistente.
Ruolo: admin

DELETE /tables/{id}
Descrizione: elimina un tavolo.
Ruolo: admin


ORDERS - ORDINI


GET /orders
Query params:
-status: cerca per status
-customer_id: da gli ordini per cliente
-data: da gli ordini per quella data
Descrizione: ottiene tutti gli ordini.
Ruolo: admin, waiter

GET /orders/{id}
Descrizione: ottiene un singolo ordine tramite id.
Ruolo: admin, waiter

GET /orders/table/{tableId}
Query params:
-data
Descrizione: ottiene gli ordini associati a un tavolo.
Ruolo: admin, waiter

POST /orders
Descrizione: crea un nuovo ordine.
Ruolo: customer, admin, waiter

PUT /orders/{id}
Descrizione: modifica un ordine esistente.
Ruolo: admin, waiter

PUT /orders/{id}/status
Descrizione: aggiorna lo stato di un ordine.
Ruolo: admin, waiter

DELETE /orders/{id}
Descrizione: elimina un ordine.
Ruolo: admin


ASSISTANCE REQUESTS - RICHIESTE DI ASSISTENZA

GET /assistance-requests
Descrizione: ottiene tutte le richieste di assistenza.
Ruolo: admin, waiter

GET /assistance-requests/{id}
Descrizione: ottiene una singola richiesta di assistenza tramite id.
Ruolo: admin, waiter

POST /assistance-requests
Descrizione: crea una nuova richiesta di assistenza.
Ruolo: customer


PUT /assistance-requests/{id}/status
Descrizione: aggiorna lo stato di una richiesta di assistenza.
Ruolo: admin, waiter


USER AREA - AREA UTENTE


GET /me
Descrizione: ottiene le informazioni dell'utente autenticato.
Ruolo: utente autenticato

GET /me/orders
-status: cerca per status
-data: da gli ordini per quella data
Descrizione: ottiene gli ordini dell'utente autenticato.
Ruolo: customer

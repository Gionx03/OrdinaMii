
Formato usato:
METODO  /endpoint
Descrizione: cosa fa l'endpoint
Ruolo: ruolo che può usare l'endpoint


DISHES - PIATTI / MENU


GET /dishes
Descrizione: ottiene tutti i piatti oppure li filtra tramite query param.
Query param:
- category: filtra per categoria.
- name: cerca per nome.
-descr: cerca per la descrizione, che contiene ingredienti
Ruolo: tutti

GET /dishes/{id}
Descrizione: ottiene un singolo piatto tramite id.
Ruolo: tutti

POST /dishes
Descrizione: crea un nuovo piatto.
Ruolo: admin/staff

PUT /dishes/{id}
Descrizione: modifica un piatto esistente.
Ruolo: admin/staff

DELETE /dishes/{id}
Descrizione: elimina un piatto.
Ruolo: admin/staff


TABLES


GET /tables

Ruolo: tutti


POST /tables

Ruolo: admin

PUT /tables/{id}
Descrizione: modifica un tavolo esistente.
Ruolo: admin

DELETE /tables/{id}
Descrizione: elimina un tavolo.
Ruolo: admin


ORDERS - ORDINI


GET /orders
Query params:
-status: cerca per status
-customer_id: da gli ordini per cliente
-data: da gli ordini per quella data
Descrizione: ottiene tutti gli ordini.
Ruolo: admin, waiter

GET /orders/{id}
Descrizione: ottiene un singolo ordine tramite id.
Ruolo: admin, waiter


POST /orders
Descrizione: crea un nuovo ordine.
Ruolo: customer, admin, waiter

PUT /orders/{id}
Descrizione: modifica un ordine esistente.
Ruolo: admin, waiter

PUT /orders/{id}/status
Descrizione: aggiorna lo stato di un ordine.
Ruolo: admin, waiter

DELETE /orders/{id}
Descrizione: elimina un ordine.
Ruolo: admin

USER AREA


GET /me
Descrizione: ottiene le informazioni dell'utente autenticato.
Ruolo: utente autenticato

GET /me/orders
-status: cerca per status
-data: da gli ordini per quella data
Descrizione: ottiene gli ordini dell'utente autenticato.
Ruolo: customer


RESERVATIONS


GET /reservations
Query params:
- status: cerca le prenotazioni per stato
- user_id: cerca le prenotazioni di un utente
- table_id: cerca le prenotazioni di un tavolo
- data: cerca le prenotazioni per una determinata data
Descrizione: ottiene tutte le prenotazioni oppure le filtra tramite query param.
Ruolo: admin, waiter


GET /reservations/{id}
Descrizione: ottiene una singola prenotazione tramite id.
Ruolo: admin, waiter


GET /reservations/table/{tableId}
Query params:
- data: cerca le prenotazioni di quel tavolo per una determinata data
Descrizione: ottiene le prenotazioni associate a un tavolo.
Ruolo: admin, waiter


POST /reservations
Descrizione: crea una nuova prenotazione.
Ruolo: customer, admin, waiter


PUT /reservations/{id}
Descrizione: modifica una prenotazione esistente.
Ruolo: admin, waiter


PUT /reservations/{id}/status
Descrizione: aggiorna lo stato di una prenotazione.
Ruolo: admin, waiter


DELETE /reservations/{id}
Descrizione: elimina una prenotazione.
Ruolo: admin

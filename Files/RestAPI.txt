
Formato usato:
METODO  /endpoint
Descrizione: cosa fa l'endpoint
Ruolo: ruolo che può usare l'endpoint


DISHES - PIATTI / MENU


GET /dishes
Descrizione: ottiene tutti i piatti oppure li filtra tramite query param.
Query param:
- category: filtra per categoria.
- name: cerca per nome.
-descr: cerca per la descrizione, che contiene ingredienti
- available: filtra i piatti disponibili o non disponibili. Di default true.
Ruolo: tutti

GET /dishes/{id}
Descrizione: ottiene un singolo piatto tramite id.
Ruolo: tutti

POST /dishes
Descrizione: crea un nuovo piatto.
Ruolo: admin/staff

PUT /dishes/{id}
Descrizione: modifica un piatto esistente.
Ruolo: admin/staff

DELETE /dishes/{id}
Descrizione: disattiva un piatto impostando available=false. Il piatto resta nel database per mantenere lo storico degli ordini.
Ruolo: admin/staff


TABLES


GET /tables
- active: filtra i tavoli attivi o disattivati. Di default true.
Descrizione: ottiene tutti i tavoli del ristorante.
Ruolo: tutti


POST /tables
Descrizione: crea un nuovo tavolo.
Ruolo: admin

PUT /tables/{id}
Descrizione: modifica un tavolo esistente.
Ruolo: admin

DELETE /tables/{id}
Descrizione: disattiva un tavolo impostando active=false. Il tavolo resta nel database per mantenere lo storico di ordini, prenotazioni e richieste di assistenza.
Ruolo: admin


ORDERS - ORDINI


GET /orders
Query params:
-status: cerca per status
-customer_id: da gli ordini per cliente
-data: da gli ordini per quella data
Descrizione: ottiene tutti gli ordini.
Ruolo: admin, waiter

GET /orders/{id}
Descrizione: ottiene un singolo ordine tramite id.
Ruolo: admin, waiter


POST /orders
Descrizione: crea un nuovo ordine per uno specifico utente. Può essere d'asporto o al tavolo.
Body:
- userId: id dell'utente.
- orderType: TAKE_AWAY oppure ON_THE_TABLE.
- tableId: obbligatorio se orderType=ON_THE_TABLE, nullo se orderType=TAKE_AWAY.
- items: lista dei piatti ordinati.
Ruolo: admin, waiter

PUT /orders/{id}
Descrizione: modifica un ordine esistente.
Ruolo: admin, waiter

PUT /orders/{id}/status
Descrizione: aggiorna lo stato operativo di un ordine.
Body:
- status: PENDING, PREPARING, SERVED, CANCELLED.
Ruolo: admin, waiter, cook

PUT /orders/{id}/payment-status
Descrizione: aggiorna lo stato del pagamento di un ordine.
Body:
- paymentStatus: NOT_PAID, PENDING, PAID, PAY_AT_COUNTER, CANCELLED.
Ruolo: admin, waiter



DELETE /orders/{id}
Descrizione: cancella logicamente un ordine impostando status=CANCELLED e paymentStatus=CANCELLED.
L'ordine e le righe restano nel database per mantenere lo storico.
Ruolo: admin

PERSONAL AREA

GET /me
Descrizione: ottiene le informazioni dell'utente autenticato.
Ruolo: authenticated

GET /me/orders
Query params:
- status
- startDate
- page
- size
- sort
Descrizione: ottiene gli ordini dell'utente autenticato, eventualmente filtrati per status e data.
Ruolo: customer

GET /me/reservations
Query params:
- startDate
- page
- size
- sort
Descrizione: ottiene le prenotazioni dell'utente autenticato da una certa data.
Ruolo: customer

PATCH /me
Descrizione: modifica parzialmente il profilo dell'utente autenticato, ad esempio il numero di telefono.
Ruolo: customer

POST /me/orders
Descrizione: crea un ordine per l'utente autenticato. Lo userId viene ricavato dal JWT e non viene passato nel body.
Body:
- orderType: TAKE_AWAY oppure ON_THE_TABLE.
- tableId: obbligatorio se orderType=ON_THE_TABLE, nullo se orderType=TAKE_AWAY.
- items: lista dei piatti ordinati.
Ruolo: customer


POST /me/reservations
Descrizione: crea una prenotazione per l'utente autenticato. Lo userId viene ricavato dal JWT e non viene passato nel body.
Body:
- date: data della prenotazione.
- time: ora della prenotazione.
- numberOfPeople: numero di persone.
- tableId: id del tavolo.
Ruolo: customer

RESERVATIONS


GET /reservations
Query params:
- status: cerca le prenotazioni per stato
- user_id: cerca le prenotazioni di un utente
- table_id: cerca le prenotazioni di un tavolo
- data: cerca le prenotazioni per una determinata data
Descrizione: ottiene tutte le prenotazioni oppure le filtra tramite query param.
Ruolo: admin, waiter


GET /reservations/{id}
Descrizione: ottiene una singola prenotazione tramite id.
Ruolo: admin, waiter


GET /reservations/table/{tableId}
Query params:
- data: cerca le prenotazioni di quel tavolo per una determinata data
Descrizione: ottiene le prenotazioni associate a un tavolo.
Ruolo: admin, waiter


POST /reservations
Descrizione: crea una nuova prenotazione per uno specifico utente.
Body:
- userId: id dell'utente.
- date: data della prenotazione.
- time: ora della prenotazione.
- numberOfPeople: numero di persone.
- tableId: id del tavolo.
Ruolo: admin, waiter


PUT /reservations/{id}
Descrizione: modifica una prenotazione esistente.
Ruolo: admin, waiter


PUT /reservations/{id}/status
Descrizione: aggiorna lo stato di una prenotazione.
Ruolo: admin, waiter


DELETE /reservations/{id}
Descrizione: cancella logicamente una prenotazione impostando status=CANCELLED.
La prenotazione resta nello storico e non blocca più il tavolo.
Ruolo: admin
